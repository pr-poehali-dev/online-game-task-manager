# Обновление от 16 июля — как перенести на свой сервер

В этой папке лежат **актуальные версии всех файлов**, изменённых сегодняшними правками
(с сохранением исходного пути — папку `update/` при копировании просто "накладывайте"
поверх своего проекта, убрав сам префикс `update/`).

Также в корне лежит `changes.patch` — единый git-патч со всеми изменениями, если удобнее
накатить через `git apply` или `patch -p1`, а не копировать файлы руками.

---

## 0. Порядок действий (коротко)

1. Сделайте бэкап своего сервера (файлы + БД) — на всякий случай.
2. Примените 2 новые миграции БД (раздел 2).
3. Скопируйте изменённые файлы (раздел 3) поверх своих — **кроме `func2url.json`**, его у вас нет и не должно быть.
4. Проверьте `.env` — секреты не менялись, но добавлять новые не требуется, всё уже было настроено под ваш MinIO.
5. Пересоберите фронтенд (`npm run build`) и перезапустите backend-сервис.

## 1. Что нового появилось (в двух словах)

- **Файлы и вложения**: в базе знаний, задачах и идеях теперь можно прикреплять любые файлы
  (документы, архивы и т.д.), не только картинки — всё хранится в вашем MinIO. Лимит — 300 МБ на файл.
- **Админка → «Файлы»**: новый раздел, где видно все залитые файлы по разделам
  (база знаний / идеи / задачи / задачи в архиве), с размером, ссылкой на источник и кнопкой удаления.
- **Сессия входа сокращена до 24 часов** (было 30 дней), но автоматически продлевается,
  пока человек активен в приложении.
- **Рефакторинг** (без изменения поведения): `KnowledgeBase.tsx` и `Ideas.tsx` разбиты
  на более мелкие компоненты — так проще их поддерживать дальше. Работать всё должно
  ровно так же, как и раньше.
- **Раздел «К рестарту»**: задачи теперь группируются отдельными блоками по серверам
  (С4х1 / HFx3 old / HF new), а не одним общим списком. Пустые серверы не показываются.
- **Кнопка «Готово» в «К рестарту»** стала нейтральной (серой) вместо зелёной —
  теперь визуально понятно, что это действие, а не уже проставленный статус.
  Зелёным подсвечивается только при наведении и после того, как задача отмечена готовой.
- `ARCHITECTURE.md` — файл-справка по архитектуре проекта (можно не переносить, это просто документация).

## 2. Миграции базы данных (применить ДО заливки кода)

```bash
cd /var/www/era/deploy   # или ваш путь
DATABASE_URL="postgresql://era_user:ВАШ_ПАРОЛЬ@localhost:5432/era_db" \
MAIN_DB_SCHEMA="ВАША_СХЕМА" \
bash apply_migrations.sh
```

Это применит (среди прочих, уже применённых) две новые миграции:
- `db_migrations/V0030__add_kb_attachments.sql` — добавляет вложения статьям базы знаний
- `db_migrations/V0031__add_attachments_to_tasks_and_ideas.sql` — добавляет вложения задачам и идеям

⚠️ Как и в прошлый раз — обязательно передавайте `MAIN_DB_SCHEMA`, иначе новые колонки
попадут в схему `public`, а не в вашу рабочую, и вложения не будут отображаться.

## 4. Если удобнее накатить патчем вместо копирования файлов

В корне репозитория на вашем сервере:

```bash
git apply --check update/changes.patch   # проверка, что патч ляжет без конфликтов
git apply update/changes.patch           # применить
```

Если у вас не git-репозиторий (просто папка с кодом) — используйте `patch`:

```bash
cd /var/www/era   # корень вашего проекта
patch -p1 < update/changes.patch
```

Если возникнут конфликты (например, вы что-то меняли в этих же файлах) —
надёжнее скопировать файлы из раздела 3 руками, они самодостаточны и не зависят от патча.

```bash

cd backend/admin && pip install -r requirements.txt && cd ../..
cd backend/ideas && pip install -r requirements.txt && cd ../..
cd backend/knowledge && pip install -r requirements.txt && cd ../..
cd backend/tasks && pip install -r requirements.txt && cd ../..
cd backend/tg-webhook && pip install -r requirements.txt && cd ../..
cd backend/auth && pip install -r requirements.txt && cd ../..

systemctl restart era-backend   # или как называется ваш сервис
```

## 6. Проверка после обновления

- [ ] В базе знаний / задаче / идее можно прикрепить произвольный файл (не картинку) — он появляется в MinIO
- [ ] В админке появилась кнопка «Файлы», открывается список по 4 разделам
- [ ] Клик по названию источника файла в этом списке открывает нужную статью/идею/задачу
- [ ] Удаление файла в админке убирает его и из списка, и из MinIO
- [ ] Вход в приложение работает, сессия не обрывается посреди работы (автопродление)

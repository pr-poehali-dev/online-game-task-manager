# Обновления от 17 июля

Три независимых изменения. Миграции БД и backend не затронуты — правки только во фронтенде.

## 1. Фикс раздела «К рестарту»

### Что было не так
В разделе «К рестарту» задачи группируются по серверам. Список серверов для группировки
был жёстко задан (3 конкретных сервера). Если у задачи стоял сервер, которого нет в этом
списке (например, изменённый/дополнительный сервер в вашей команде), задача пропадала из
раздела визуально — при этом счётчик задач наверху продолжал считать её корректно.
Из-за этого выглядело так, будто «счётчик есть, а список пуст».

### Что исправлено
Теперь для серверов, не входящих в основной список, автоматически создаётся отдельный
блок группировки — ни одна задача больше не потеряется из вида.

### Файл для переноса
```
src/pages/index/Restart.tsx
```

## 3. Фикс удаления задачи с доски

### Что было не так
При нажатии «Удалить» на карточке задачи (полное удаление, не архивирование) задача
сразу пропадала с экрана, но реальный запрос на удаление в базе данных отправлялся
с задержкой в 5 секунд (это было сделано ради кнопки «Восстановить» в уведомлении).
Если пользователь обновлял страницу раньше, чем проходили эти 5 секунд, запрос на
удаление так и не успевал уйти на сервер — задача оставалась в базе и снова появлялась
после перезагрузки страницы.

### Что исправлено
Запрос на удаление теперь отправляется на сервер сразу же. Кнопка «Восстановить»
в уведомлении по-прежнему работает — при нажатии задача создаётся заново.

### Файл для переноса
```
src/pages/index/useTaskActions.ts
```

## Как перенести

Проще всего — скопировать перечисленные файлы из этой папки поверх файлов на сервере,
сохранив пути (`index.html` в корень, остальное — в `public/`).

Либо применить патч (только для текстовых изменений в `index.html` и `Restart.tsx`,
картинки патчем не переносятся — их нужно скопировать отдельно, они уже лежат готовыми
в папке `public/` рядом с этим README):

```bash
cd /var/www/era   # корень вашего проекта
git apply update/changes.patch
# либо, если это не git-репозиторий:
patch -p1 < update/changes.patch
```

## После переноса
```bash
npm run build
```
Backend перезапускать не нужно.

## Проверка

- Удалите тестовую задачу с доски и сразу обновите страницу — задача не должна вернуться.

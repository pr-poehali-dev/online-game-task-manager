# Точечный фикс от 17 июля — раздел «К рестарту»

Один файл, один баг. Миграции БД и backend не затронуты.

## Что было не так
В разделе «К рестарту» задачи группируются по серверам. Список серверов для группировки
был жёстко задан (3 конкретных сервера). Если у задачи стоял сервер, которого нет в этом
списке (например, изменённый/дополнительный сервер в вашей команде), задача пропадала из
раздела визуально — при этом счётчик задач наверху продолжал считать её корректно.
Из-за этого выглядело так, будто «счётчик есть, а список пуст».

## Что исправлено
Теперь для серверов, не входящих в основной список, автоматически создаётся отдельный
блок группировки — ни одна задача больше не потеряется из вида.

## Файл для переноса
```
src/pages/index/Restart.tsx
```

Замените этот файл целиком на сервере, либо примените патч:

```bash
cd /var/www/era   # корень вашего проекта
git apply update/changes.patch
# либо, если это не git-репозиторий:
patch -p1 < update/changes.patch
```

## После переноса
```bash
npm run build
```
Backend перезапускать не нужно — правка только во фронтенде.

## Проверка
- Откройте вкладку «К рестарту» — задача с любым сервером теперь отображается в своём блоке.

## 2. Диагностика: не пишутся патчноуты при закрытии задачи «К рестарту»

### Контекст
Запись в журнал патчноутов (`backend/tasks/index.py`, экшен `archive`)
создаётся автоматически **только** если одновременно выполняются три условия:
1. задача была в колонке `restart` («К рестарту»);
2. при отправке в архив выбран исход `done` («Реализовано»);
3. у задачи заполнено поле `server`.

Если патчноуты не появляются — вероятнее всего, либо у задачи не указан
сервер, либо на сервере не применена миграция `V0045`, создающая саму таблицу
`patchnotes`.

### SQL-запросы для проверки на своей базе

Выполняются через `psql "$DATABASE_URL" -c "..."` на self-hosted сервере.

**1. Проверить, что таблица `patchnotes` существует** (миграция V0045 применена):
```sql
SELECT EXISTS (
  SELECT 1 FROM information_schema.tables
  WHERE table_name = 'patchnotes'
) AS table_exists;
```
Если `table_exists = false` — нужно применить миграции:
```bash
cd /var/www/era
DATABASE_URL="postgresql://era_user:пароль@localhost:5432/era_db" \
MAIN_DB_SCHEMA="public" \
bash deploy/apply_migrations.sh
```

**2. Посчитать, сколько записей уже в журнале патчноутов:**
```sql
SELECT COUNT(*) FROM patchnotes;
```

**3. Найти задачи-кандидаты на «К рестарту» и проверить, заполнен ли у них сервер:**
```sql
SELECT id, title, server, column_id, outcome, archived
FROM tasks
WHERE column_id = 'restart' OR outcome = 'done'
ORDER BY id DESC
LIMIT 20;
```
Если в колонке `server` пусто (`NULL`) у задач, которые были в `restart` и
закрыты с исходом `done` — вот и причина: без сервера запись не создаётся.
Нужно открыть такую задачу и вручную указать сервер перед архивацией.

### Проверка
- Выполните запрос №1 — таблица должна существовать.
- Откройте карточку задачи из раздела «К рестарту», убедитесь, что в ней
  выбран сервер, отметьте «Готово» → «В архив» → «Реализовано».
- Выполните запрос №2 до и после — счётчик должен увеличиться на 1.

## 3. Список команды в боковой панели больше не уезжает вниз

### Что было не так
Когда задач в категориях было слишком много, вся боковая панель растягивалась
вместе со страницей, и блок «Команда» (прижатый к низу) уезжал далеко за
пределы экрана — приходилось скроллить всю страницу, чтобы его увидеть.

### Что исправлено
Теперь боковая панель занимает ровно высоту экрана. Список категорий
скроллится отдельно и независимо, а блок «Команда» всегда закреплён внизу
и виден на экране. То же самое исправлено и в мобильном меню.

### Файлы для переноса
```
src/pages/Index.tsx
src/pages/index/IndexSidebar.tsx
src/pages/index/IndexTopbar.tsx
```

### Проверка
- Откройте доску с большим количеством категорий/задач.
- Растяните окно браузера по высоте — список «Команда» должен оставаться
  на месте внизу боковой панели, не уезжая за пределы экрана.
- Проверьте то же самое в мобильном меню (гамбургер-иконка).

## 4. Меню выбора исхода задачи в «К рестарту» больше не перекрывается

### Что было не так
В разделе «К рестарту» при клике на кнопку «В архив» у выполненной задачи
открывалось выпадающее меню «Исход задачи» — но если рядом (справа снизу)
была соседняя карточка, она визуально перекрывала это меню, и часть пунктов
было не видно и не нажать.

### Что исправлено
Карточка с открытым меню теперь поднимается выше соседних карточек по слоям,
а само меню получило более высокий приоритет отображения — оно больше не
прячется под другими карточками.

### Файлы для переноса
```
src/pages/index/Restart.tsx
```

### Проверка
- В разделе «К рестарту» отметьте задачу «Готово», нажмите «В архив» —
  меню «Исход задачи» должно полностью отображаться поверх соседних карточек.

## 5. Кнопка «Ответить» теперь доступна у всех комментариев

### Что было не так
В комментариях к задачам и идеям кнопка «Ответить» показывалась только у
комментариев верхнего уровня — ответить на уже существующий ответ было
невозможно.

### Что исправлено
Кнопка «Ответить» теперь есть у любого комментария, включая ответы. При этом
структура обсуждения остаётся плоской (без глубокой вложенности) — ответ на
ответ технически прикрепляется к тому же родительскому комментарию верхнего
уровня.

### Файлы для переноса
```
src/pages/index/Ideas.tsx
src/pages/index/TaskComments.tsx
src/pages/index/ideas/IdeaComments.tsx
```

### Проверка
- Ответьте на комментарий к задаче, затем ответьте на этот же ответ —
  кнопка «Ответить» должна быть доступна в обоих случаях, а новый комментарий
  появится в общей ленте ответов под исходным комментарием верхнего уровня.
- Проверьте то же самое в комментариях к идеям.

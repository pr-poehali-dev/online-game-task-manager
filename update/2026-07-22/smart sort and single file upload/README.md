# Умная сортировка задач + загрузка одиночных файлов в патчи

Эта папка — дополнение к сегодняшнему обновлению (`update/2026-07-22`).
Содержит два независимых улучшения из двух последних коммитов.

---

## 1. Умная сортировка задач на доске

### Что изменилось
Раньше сортировка на доске по умолчанию отсутствовала («По умолчанию» —
без сортировки, задачи шли в произвольном порядке хранения). Теперь
вместо неё включена **умная сортировка**, которая работает по умолчанию
сразу при открытии доски:

1. **Наверху** — критичные и высокоприоритетные задачи, а также любые
   задачи, у которых дедлайн уже истёк или наступает в течение суток.
2. **Внутри этой группы** — сортировка по срочности дедлайна (у кого
   раньше горит срок — выше).
3. **Остальные задачи** — по дате создания, старые сверху, чтобы не
   терялись в бэклоге.

Прежние варианты сортировки («По приоритету», «Сначала новые», «Сначала
старые») остались доступны в том же выпадающем меню — умная сортировка
добавлена как замена пункта «По умолчанию».

### Файл для переноса
```
src/pages/index/Board.tsx
```

### Проверка после переноса
- Откройте доску — в колонках задачи должны быть отсортированы: сверху
  критичные/горящие, затем остальные по дате создания (старые выше).
- В меню сортировки (кнопка справа сверху над доской) должен появиться
  пункт «Умная сортировка» вместо «По умолчанию».

## 2. Загрузка одиночного файла в любую папку патча + кнопка «Скачать всё»

### Что изменилось
1. **Одиночный файл без папки** теперь можно перетащить в **любую**
   папку дерева патча — не только в корневую (как раньше), но и в любую
   вложенную папку на любом уровне. Раньше drag&drop работал только на
   корневые папки верхнего уровня.
2. Добавлена кнопка **«Скачать всё»** рядом с названием сервера — по
   нажатию собирает и скачивает архив (zip) сразу из всего дерева файлов
   выбранного сервера (появляется, только если в дереве есть хотя бы
   один файл).

Текст встроенной инструкции («Как работать с патчами») на странице
тоже обновлён с учётом этих изменений.

### Файлы для переноса
```
backend/patches/index.py
backend/patches/tests.json
src/pages/index/Patches.tsx
src/pages/index/PatchesTreeFolder.tsx
```

Файл `backend/patches/index.py` уже адаптирован под собственное
S3/MinIO-хранилище (без жёстко зашитого облачного адреса poehali.dev) —
адрес хранилища берётся из переменных `S3_ENDPOINT`, `S3_PUBLIC_URL`
(при необходимости `CDN_BASE_URL`), с запасным вариантом на локальный
MinIO по адресу `http://127.0.0.1:9000`.

⚠️ Если вы уже применяли обновления из `update/2026-07-21/fix patch
loading` — новый `backend/patches/index.py` нужно скопировать **поверх**
него, он содержит все прошлые изменения (фикс загрузки файлов) плюс это
новое действие `zip_all`.

### Проверка после переноса
- Откройте раздел «Патчи», перетащите одиночный файл (без папки) на
  любую вложенную папку в дереве — файл должен загрузиться именно туда.
- Рядом с названием сервера (там, где счётчик файлов) должна появиться
  кнопка «Скачать всё» — нажмите её и убедитесь, что скачивается архив
  со всеми файлами дерева этого сервера.

```bash
cp "update/2026-07-22/smart sort and single file upload/src/pages/index/Board.tsx" src/pages/index/Board.tsx
cp "update/2026-07-22/smart sort and single file upload/backend/patches/index.py" backend/patches/index.py
cp "update/2026-07-22/smart sort and single file upload/backend/patches/tests.json" backend/patches/tests.json
cp "update/2026-07-22/smart sort and single file upload/src/pages/index/Patches.tsx" src/pages/index/Patches.tsx
cp "update/2026-07-22/smart sort and single file upload/src/pages/index/PatchesTreeFolder.tsx" src/pages/index/PatchesTreeFolder.tsx

npm install
npm run build
sudo systemctl restart era-backend
```

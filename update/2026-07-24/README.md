## 1. Приватные сообщения (спойлеры) в описании и комментариях

### Что это
В описании задачи (кнопка с перечёркнутым глазом рядом с полем «Описание»,
видна только в режиме редактирования) и при написании нового комментария
(такая же иконка рядом с кнопкой отправки) можно оставить текст, видимый
только выбранному адресату — и тем, у кого есть право «Просмотр чужих
приватных сообщений» (см. пункт 2).

Приватное сообщение отображается прямо в теле описания/комментария как
спойлер: по умолчанию скрыто и подписано «Приватно для Имени — нажмите,
чтобы показать», по клику текст раскрывается.

### Важный технический момент
Текст приватного сообщения хранится и передаётся с сервера в браузер
**только** тем, кому положено — автору, адресату и обладателям права
просмотра чужих сообщений. Это не визуальная маскировка: скрытый текст
физически не долетает до чужого браузера, поэтому его нельзя достать через
код страницы, как было бы при обычном CSS-спойлере.

### Где именно доступны кнопки
- **Кнопка добавления** («глазик»): в описании задачи — только в режиме
  редактирования; у нового комментария — всегда, у уже написанных
  комментариев — рядом с каждым комментарием (кнопка «Приватная заметка»).
- **Крестик удаления**: в описании задачи — только в редакторе (в режиме
  просмотра заметку можно только прочитать, не удалить); у комментариев —
  без изменений, как и раньше.
- Удалить сообщение может автор или администратор.

### Проверка

  рядом с «Описание» — должно открыться окно выбора адресата и текста.
  После добавления заметка сразу видна под редактором как спойлер.
- Выйдите из режима редактирования — кнопки добавления и крестика удаления

## 2. Право «Просмотр чужих приватных сообщений»

Новое индивидуальное право (раздел «Приватные сообщения» в дереве прав
участника, значок ключа в управлении командой). Раньше приватные сообщения

это право можно выдать или забрать любому участнику, включая администраторов —

## 3. Бейдж «Требуется заливки в лаунчер» + уведомления

### Что это
Бейдж появляется на карточке задачи, если к ней прикреплён хотя бы один файл
патча (раздел «Патчи») и задача при этом в состоянии, готовом к раскатке —
колонка «К рестарту» или статус деплоя «Можно заливать на лайв» — но ещё не
отмечена как загруженная. Кнопка «Отметить как загружено в лаунчер» в карточке
задачи убирает бейдж (заменяет на зелёный «Загружено в лаунчер»).

Если после отметки статус деплоя или колонка задачи снова меняются — отметка
«Загружено» автоматически сбрасывается.

### Уведомления
Как только бейдж у задачи появляется (смена статуса/колонки на подходящий,
снятие отметки «Загружено», прикрепление файла патча к уже готовой задаче),
все участники с новым правом «Уведомление о лаунчере» получают уведомление —
в приложении (колокольчик) и в Telegram.

### Файлы для переноса
```

backend/patches/index.py
backend/tasks/index.py
src/lib/auth.tsx
src/pages/admin/adminShared.ts
src/pages/index/NotificationBell.tsx
db_migrations/V0054__add_launcher_uploaded_to_tasks.sql
```

### Проверка
- Прикрепите файл патча к задаче в статусе «Можно заливать на лайв» —
  на карточке должен появиться красный пульсирующий бейдж.
- Нажмите «Отметить как загружено» — бейдж должен смениться на зелёный.
- Участнику с правом «Уведомление о лаунчере» должно прийти уведомление
  в колокольчик и (если вход через Telegram-бота) сообщение в Telegram.

## 4. Умная сортировка задач на доске

### Что изменилось
Режим сортировки по умолчанию на доске сменился с «По умолчанию» на «Умная
сортировка» (иконка со звёздочками в меню сортировки). Сначала показываются
самые горящие задачи — с высоким/критическим приоритетом и/или дедлайном,
который уже истёк либо наступает в течение суток; внутри этой группы — по
сроку дедлайна (раньше срок — выше). Остальные задачи идут по дате создания
(старые выше, чтобы не терялись в бэклоге).

### Файлы для переноса
```

```

### Проверка
- Откройте доску — в меню сортировки (иконка) должен быть выбран пункт
  «Умная сортировка» по умолчанию.
- Задачи с критическим приоритетом или горящим дедлайном должны быть

## 5. Декомпозиция `TaskModal.tsx`

### Что изменилось
Один большой файл `TaskModal.tsx` (было ~510 строк) разбит на 4 файла без
изменения логики или разметки:
- `TaskModalHeader.tsx` — шапка модалки (бейджи, кнопка «Редактировать»,

  (приоритет/сервер/категория/исполнители/спринт/дедлайн/статьи базы знаний).

  лаунчера, статус деплоя, ссылки.
- `TaskModal.tsx` — состояние и обработчики, собирает остальные компоненты
  + блок комментариев и футер.

Чисто техническое изменение для удобства дальнейшей поддержки — поведение
интерфейса не меняется.

## 7. Разное: фиксы и мелкие улучшения

- **Восстановление задачи в Done из «К рестарту»** — добавлена возможность
  вернуть задачу обратно в колонку Done, если её отправили в рестарт по
  ошибке (`backend/tasks/index.py`, `src/pages/index/Restart.tsx`,
  `src/pages/index/useTaskActions.ts`, `src/pages/Index.tsx`,
  `src/pages/index/IndexMain.tsx`).
- **Открытие статьи базы знаний при смене статуса деплоя** — если статья
  была открыта поверх задачи и пользователь меняет статус деплоя, статья
  остаётся открытой, а не закрывается неожиданно (`src/pages/index/useDeepLinks.ts`).
- **Подсветка кнопки «Показать файлы патча этой задачи»** — кнопка теперь
  окрашивается акцентным цветом, если у задачи есть прикреплённые файлы
  патча, чтобы это было заметно с первого взгляда (входит в `TaskModalDetails.tsx`).
- **Карточки в разделе «Патчи» не мигают** — исправлено лишнее перерисовывание
  дерева файлов при обновлении списка (`backend/patches/index.py`).
- **Бейдж лаунчера не остаётся после переноса в рестарт** — исправлена гонка
  состояний, из-за которой бейдж «Требуется заливки» иногда продолжал
  показываться на карточке уже после того, как задача была перенесена
  (`backend/tasks/index.py`).
- **Запоминание выбранной вкладки в «Патчах»** — при повторном открытии
  раздела остаётся выбранным последний просмотренный сервер
  (`src/pages/Index.tsx`, `src/pages/index/IndexMain.tsx`,
  `src/pages/index/Patches.tsx`).
- **Автораскрытие папок с файлами патча** — папки в дереве, где есть хотя бы
  один файл, теперь разворачиваются автоматически при открытии дерева
  (`src/pages/index/PatchesTreeFolder.tsx`).
- **Сервер по умолчанию для нового спринта** — при создании задачи спринт
  по умолчанию не выбран, вместо случайного первого варианта
  (`src/pages/index/CreateTaskModal.tsx`).
- **Загрузка данных доски** — доработки в `useBoardData.ts` для поддержки
  бейджа лаунчера (список задач с прикреплёнными файлами патча).
